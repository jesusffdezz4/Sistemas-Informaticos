Software de código abierto

En 1998, OSI nació como una derivación más pragmática de software que defiende no tanto aspectos filosóficos como tecnológicos. En los aspectos puramente funcionales se pueden considerar equivalentes. Los requisitos para que OSI considere el software de código abierto son los expresados en este decálogo:

  1. Libertad de uso sin discriminación de grupos o individuos.
  2. Libertad de uso sin discriminación de áreas de iniciativa.
  3. Un programa de código abierto no se puede licenciar únicamente como parte indisoluble de otro.
  4. La licencia no debe restringir la licencia de otro software, es decir, si el software se distribuye junto con otros, no puede obligar a estos otros a ser también de código abierto.
  5. La licencia debe ser tecnológicamente neutral.
  6. Disponibilidad del código fuente.
  7. Libre redistribución de software.
  8. Los derechos de distribución de la licencia deben aplicarse a todos los que los redistribuyan sin requerir una licencia adicional.
  9. Libertad de generar obras derivadas,
  10. Aunque la redistribución de estos trabajos derivados puede limitarse a meros parches que conserven la licencia original.

Este decálogo desarrolla desde otra perspectiva las cuatro libertades de software en las que se basa la filosofía del software libre.

Aviso: el código abierto no debe confundirse con un software que simplemente muestra su código fuente.
