Software libre

Ideado por la FSF, cumple las cuatro libertades (relacionadas con los cuatro aspectos ya esbozados):

  1. Libertad de uso, es decir, sin limitaciones en quién usa el software ni para qué lo usa.
  2. Libertad de acceso al código fuente, es decir, el usuario tiene derecho a acceder al código escrito por el programador y a hacer los cambios que considere oportunos para ajustarlo a sus necesidades.
  3. Libertad de distribución, es decir, el usuario tiene total libertad para copiar el software para terceros.
  4. Libertad de mejora, es decir, el usuario puede publicar cualquier cambio que haya hecho en el código fuente para que terceros puedan beneficiarse de él.

Hay dos familias numerosas de licencias libres dependiendo de cómo articulen la última de las libertades:

  • Licencias de tipo BSD, en las que no se establecen restricciones en el nuevo software generado gracias a la libertad de mejora. Recibe este nombre porque fue la licencia con la que se distribuyó la versión de UNIX desarrollada por la Universidad de Berkeley y que recibió el nombre de BSD. Las licencias de este tipo son:
    • Licencia BSD utilizada por algunas distribuciones derivadas del BSD UNIX original como FreeBSD o NetBSD.
    • Licencia MIT, creada en el MIT para licenciar el servidor gráfico X11. Es la licencia más utilizada en Github.
    • Licencia Apache, que es la licencia utilizada por la Fundación Apache para sus productos.
    • Licencia ISC utilizada por el ISC para sus desarrollos (por ejemplo, el servidor de enlace DNS) o por la distribución OpenBSD.
    • Licencia Beerware, licencia extremadamente permisiva que solo anima al usuario a comprar una cerveza al desarrollador si alguna vez lo encuentran.
  • Las licencias tipo GPL (o licencias copyleft), que prescriben que cualquier software derivado debe cumplir necesariamente las cuatro libertades y, por lo tanto, volver a ser GPL. Esta obligación de mantener la licencia es lo que se llama copyleft. Es la licencia que la FSF adopta en sus desarrollos para el proyecto GNU y a la que se aloja una gran cantidad de software libre, aunque no pertenezca a este proyecto, como el núcleo Linux. Hay dos variantes fundamentales de la licencia GPL:
    • La Licencia LGPL, más permisiva que la GPL normal y que permite el uso del software con esta licencia en un proyecto que no es gratuito si está ligado dinámicamente. Es adecuada para crear bibliotecas.
    • La Licencia AGPL, que es más restrictiva que la GPL normal, requiere que el usuario entregue el código fuente, aunque no tenga que proporcionar una copia para usarlo. Un ejemplo típico es el de un software de servidor: un software de servidor no tiene que ser suministrado al cliente, si montamos el servidor nosotros mismos.
